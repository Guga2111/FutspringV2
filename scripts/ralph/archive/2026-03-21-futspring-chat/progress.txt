## Codebase Patterns
- Backend uses Lombok @Data, @Builder, @NoArgsConstructor, @AllArgsConstructor on all entities and DTOs
- DTOs have a static `from(Entity entity)` factory method for entity-to-DTO conversion
- JPA entities use @CreationTimestamp from org.hibernate.annotations for createdAt fields
- ManyToMany relationships use @JoinTable with explicit join column names
- Hibernate DDL is set to `update` mode - tables auto-created from entities
- Java/Maven not available in this environment - "Typecheck passes" means code is syntactically correct Java
- Frontend TypeScript typecheck: `cd frontend && npx tsc -b --noEmit`
- Controllers inject `Authentication` and get current user email via `(String) authentication.getPrincipal()`
- Services use `@Transactional` (write) and `@Transactional(readOnly = true)` for read operations
- Lazy-loaded collections (@ManyToMany with FetchType.LAZY) are safe to access within @Transactional service methods
- Conflict responses use AppException(HttpStatus.CONFLICT, message)
- File uploads: `@RequestParam("file") MultipartFile file`; store under `@Value("${app.uploads.dir:uploads}")` dir with UUID prefix
- No browser testing tools available; manual browser verification needed
- When injecting a new service into a controller that uses @RequiredArgsConstructor, just add the final field — no @Autowired needed
- Upsert pattern: repository.findByX().orElse(new Entity()) then save()
- Date formatting for ISO date strings: append 'T12:00:00' before parsing to avoid timezone off-by-one
- Status badge colors: SCHEDULED=blue-100/800, CONFIRMED=yellow-100/800, IN_COURSE=green-100/800, FINISHED=gray-100/800, CANCELED=red-100/800
- DailyDetailDTO includes `isAdmin` boolean — check pelada.getAdmins().contains(caller) in getDailyDetail
- recharts is installed (added in stats cycle); BarChart/Bar/XAxis/YAxis/Tooltip available
- @stomp/stompjs and sockjs-client need to be installed for WebSocket chat (npm install @stomp/stompjs sockjs-client @types/sockjs-client)
- STOMP reconnect backoff: set `reconnectDelay: 0` to disable built-in, use setTimeout + retryCountRef + isDeactivatingRef for manual exponential backoff
- STOMP JWT auth: pass in connectHeaders { Authorization: 'Bearer <token>' }; validate on backend via ChannelInterceptor in preSend() for CONNECT frames
- WebSocket config: @EnableWebSocketMessageBroker; /ws endpoint with SockJS; broker on /topic; app prefix /app
- ChatController uses @MessageMapping + SimpMessagingTemplate.convertAndSend to broadcast to /topic/pelada/{id}
- GET /api/v1/peladas/:id/messages returns history sorted ascending (reverse of DB query which is desc)
- WebSocket /ws/** must be permitted in SecurityConfig to allow SockJS handshake without auth
- ChatController is a @Controller (not @RestController); uses @MessageMapping not @GetMapping/@PostMapping
- ChatService separates business logic from controller: member validation + content validation + persist + return DTO
