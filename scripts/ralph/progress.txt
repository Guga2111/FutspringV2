## Codebase Patterns
- Java/Maven not available in devcontainer (Node.js-based); typecheck is verified by code review
- Spring Boot 3.2.3 used; package root is `com.futspring.backend`
- Frontend: Vite + React 18 + TypeScript + Tailwind v3 + shadcn/ui (manual, no CLI) under `frontend/`
- Frontend path alias: `@/*` → `./src/*` (set in tsconfig.app.json and vite.config.ts)
- shadcn CLI requires network; offline → install radix-ui/react-* + cva + clsx + tailwind-merge manually
- Tailwind v3 is used (not v4) for shadcn compatibility; PostCSS config present
- Axios base client in `src/api/client.ts`; interceptors to be added in US-006
- Config via env vars with defaults in application.yml using `${ENV_VAR:default}` syntax
- GlobalExceptionHandler returns `{ status, message, timestamp }` shape via ErrorResponse
- SecurityConfig: CSRF disabled, STATELESS session — standard for all protected routes
- JJWT 0.11.5 for JWT; Bucket4j 8.7.0 for rate limiting
- DTOs in `dto/` package; entities in `entity/`; repos in `repository/`; services in `service/`; controllers in `controller/`
- UserResponseDTO.from(User) static factory pattern for entity→DTO mapping
- Hibernate ddl-auto: update — no Flyway needed, table created automatically
- Bucket4j rate limiting: `Bucket.builder().addLimit(Bandwidth.classic(N, Refill.intervally(N, Duration.ofMinutes(1))))` with ConcurrentHashMap<String, Bucket> keyed by IP
- JwtService.generateToken(userId, email): uses Jwts.builder() with setSubject(email), claims userId+email, signed with Keys.hmacShaKeyFor
- AppException(HttpStatus, message) pattern for all business errors; GlobalExceptionHandler handles it generically
- JwtAuthFilter: invalid token → write 401 in filter; missing token → passthrough → AuthenticationEntryPoint in SecurityConfig handles 401
- ObjectMapper injected as `filterChain(@Bean)` method param (Spring Boot auto-provides it)
- ESLint `react-refresh/only-export-components`: split non-component exports (context objects, cva variants) into `.ts` files; component files `.tsx` should only export components
- `useSearchParams` from react-router-dom for URL query param sync (e.g. active tab)
