## Codebase Patterns
- Backend services use `@Service` + `@RequiredArgsConstructor` + `private final` constructor injection (no @Autowired field injection)
- AppException(HttpStatus, message) is the standard error pattern
- Tests use `new ServiceClass(repo1, repo2, ...)` with Mockito `@Mock` fields тАФ no Spring context
- When adding a new service dependency, update ALL test files that `new` the affected service class
- `@Value("${app.uploads.dir:uploads}")` fields cannot coexist with `@RequiredArgsConstructor` if you want them injected тАФ use `@Value` on a non-final field instead, or move to a config bean
- File upload logic uses: 5MB size check тЖТ MIME check (jpeg/png/webp) тЖТ UUID filename тЖТ Files.copy тЖТ return filename; IOException тЖТ AppException(500, "Failed to store file")
- UserAuthenticationHelper lives in `com.futspring.backend.helper` and delegates user lookup to avoid repeating the findByEmail pattern; services that removed their userRepository dependency (ChatService, RankingService, AwardsService) drop the repo from their imports but DailyService/PeladaService retain it for other uses
- Tests that check user-not-found now mock `userAuthHelper.getAuthenticatedUser(email)` to throw `AppException(NOT_FOUND)` instead of returning `Optional.empty()` from userRepository тАФ and assert NOT_FOUND (not UNAUTHORIZED)
