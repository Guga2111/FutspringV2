## Codebase Patterns
- Frontend custom hooks live at `frontend/src/components/<feature>/hooks/<hookName>.ts`; the hooks directory is created on first use (not pre-existing)
- Custom data-fetching hooks expose individual refetch functions (e.g., `refetchPelada`, `refetchDailies`) in addition to a combined `refetch` so callers can trigger targeted refreshes
- When extracting a service, the delegation stub in the original service must @Transactional-annotate wrapper methods and call the sub-service; the original service's tests become delegation tests (mock sub-service â†’ verify call), and a new test file covers the real logic
- `DailyAttendanceService` is in the same package as `DailyService` so clearAttendees (package-private) is accessible to future orchestration code in that package
- Backend services use `@Service` + `@RequiredArgsConstructor` + `private final` constructor injection (no @Autowired field injection)
- AppException(HttpStatus, message) is the standard error pattern
- Tests use `new ServiceClass(repo1, repo2, ...)` with Mockito `@Mock` fields â€” no Spring context
- When adding a new service dependency, update ALL test files that `new` the affected service class
- `@Value("${app.uploads.dir:uploads}")` fields cannot coexist with `@RequiredArgsConstructor` if you want them injected â€” use `@Value` on a non-final field instead, or move to a config bean
- File upload logic uses: 5MB size check â†’ MIME check (jpeg/png/webp) â†’ UUID filename â†’ Files.copy â†’ return filename; IOException â†’ AppException(500, "Failed to store file")
- UserAuthenticationHelper lives in `com.futspring.backend.helper` and delegates user lookup to avoid repeating the findByEmail pattern; services that removed their userRepository dependency (ChatService, RankingService, AwardsService) drop the repo from their imports but DailyService/PeladaService retain it for other uses
- Tests that check user-not-found now mock `userAuthHelper.getAuthenticatedUser(email)` to throw `AppException(NOT_FOUND)` instead of returning `Optional.empty()` from userRepository â€” and assert NOT_FOUND (not UNAUTHORIZED)
- `DailyDTOMapper` is a `@Component` (not `@Service`) that holds `buildTeamDTO` â€” injected into both `DailyTeamManagementService` and `DailyService` (for `getDailyDetail`); use `dailyDTOMapper::buildTeamDTO` method reference, never `this::buildTeamDTO`
- `DailyTeamManagementService` and `DailyDTOMapper` live in the same `service` package as `DailyService`; `clearTeams` is package-private (no visibility modifier)
- `DailyController` injects all 4 service beans (DailyService, DailyAttendanceService, DailyTeamManagementService, DailyResultsService); `finalizeDaily` and `populateFromMessage` routes call the results sub-service then `dailyService.getDailyDetail` because the sub-service returns void
- `deleteDaily` orchestration order: clearResults â†’ clearTeams â†’ clearAttendees â†’ dailyRepository.delete (preserves referential integrity)
