## Codebase Patterns
- Backend services use `@Service` + `@RequiredArgsConstructor` + `private final` constructor injection (no @Autowired field injection)
- AppException(HttpStatus, message) is the standard error pattern
- Tests use `new ServiceClass(repo1, repo2, ...)` with Mockito `@Mock` fields — no Spring context
- When adding a new service dependency, update ALL test files that `new` the affected service class
- `@Value("${app.uploads.dir:uploads}")` fields cannot coexist with `@RequiredArgsConstructor` if you want them injected — use `@Value` on a non-final field instead, or move to a config bean
- File upload logic uses: 5MB size check → MIME check (jpeg/png/webp) → UUID filename → Files.copy → return filename; IOException → AppException(500, "Failed to store file")
